301 Redirect Strategy

Protecting the 400+ monthly organic clicks as we restructure the site.

What A 301 Redirect Does

A 301 redirect tells Google that a page has permanently moved to a new URL. It preserves the SEO ranking, incoming links, and traffic from the old URL and passes them to the new one — so we don't lose the organic authority already built up.

Why this matters here: we're cleaning up discontinued TYM models and replacing the legacy /branson-tym-tractors/ slug. Without proper redirects, Google would see broken pages, drop the rankings, and the 400+ monthly clicks would start shrinking fast.

The Plan

Map every existing URL before we touch anything. Pull the current sitemap plus Google Search Console data to identify every ranking page.

Build a redirect table pairing each old URL to its new destination — discontinued TYM models to the TYM hub, legacy Branson slugs to the new TYM slug, and so on.

Implement with Rank Math (already installed on the site) — clean, logged redirects that are easy to audit later.

Verify after launch — crawl the site, confirm every old URL resolves correctly, and monitor Search Console for errors during the first two weeks.
